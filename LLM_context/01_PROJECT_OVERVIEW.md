# E-MindBridge — Project Overview

## What Is This?

E-MindBridge is a **low-bandwidth-optimized mental health counseling web application** built for schools and organizations. It connects patients (students/clients) with licensed guidance counselors through a structured, AI-assisted workflow — from daily self-check-ins to tele-counseling sessions — while giving counselors and admins the data-driven tools needed to intervene early and effectively.

---

## Tech Stack

| Layer | Technology |
|---|---|
| Frontend | SvelteKit (SSR + client routing) |
| Backend / DB | Supabase (PostgreSQL + Auth + Edge Functions + Realtime) |
| AI Analysis | Anthropic Claude API (via Supabase Edge Functions) |
| Video Sessions | Low-bandwidth provider (e.g. Daily.co / Jitsi) |
| File Storage | Supabase Storage (profile photos) |
| Hosting | Vercel / Supabase cloud |

## User Roles

### Patient (Student / Client)
The primary end-user. Logs moods daily, writes journal entries, participates in tele-counseling sessions, and views wellness content. All their data feeds the AI analysis pipeline passively.

### Counselor (Guidance Counselor)
A licensed professional who monitors assigned patients via an AI-assisted dashboard. Reviews AI-generated MindProfile reports, submits clinical evaluations, and schedules or joins tele-counseling sessions. Has one or more declared **specialties** that the system uses for smart patient assignment.

### Admin
Manages the platform: approves or rejects counselor AI report requests, manages user accounts, monitors system-wide analytics, and curates wellness content (quotes, resources). The final gate before AI reports are generated on counselor request.

## Core Features

### 1. Daily Wellness Dashboard
- Displays a contextual wellness quote filtered by the patient's current mood and recent AI report category.
- Shows mood history trends in a simple chart.

### 2. Mood Tracking
- One emoji-based mood check-in per patient per day (`great → struggling`).
- Data feeds both the AI analysis engine and aggregate analytics snapshots.

### 3. The Sanctuary (Journal)
- Encrypted free-text journal entries.
- Supports multilingual input (English, Filipino, Ilocano, etc.).
- Queued for AI sentiment and threat analysis after each save.
- Counselors **cannot** read raw journal entries — they only see the AI-generated summary.

### 4. AI MindProfile Reports
- Generated by the Claude API via a Supabase Edge Function.
- Covers: risk level, sentiment, primary category, longitudinal patterns, emotional markers, threat detection (self-harm / suicidal ideation), and clinical recommendations.
- **Two report generation paths:**
  - **Automatic:** triggered when a journal entry is saved (passive, background).
  - **On-Request:** counselor submits a date-range report request → admin reviews → admin approves → Edge Function generates report.

### 5. Counselor Dashboard
- **Macro view:** global risk level, mood percentages, trend spikes across the counselor's patient group.
- **Micro view:** individual MindProfile with AI summary, threat flags, and evaluation form.
- Aggregate data sourced from pre-computed `analytics_snapshots`.

### 6. Tele-Counseling Sessions
- **Proactive:** patient requests a session from the dashboard.
- **Reactive:** counselor schedules a session based on AI risk flags.
- **Auto-assignment:** when a patient requests a session with no prior counselor, the system assigns the most available counselor whose specialties best match the patient's primary AI report category.
- After the first session, the patient defaults to keeping the same counselor but can request a reassignment.

### 7. Admin Report Request Gate
- Counselors submit a `report_requests` record specifying patient + date range.
- Admin reviews in their dashboard and approves or rejects.
- On approval, a Supabase Edge Function triggers the Claude API and writes the result to `ai_reports`.
- Full audit trail maintained.

### 8. Counselor Specialties
- Counselors declare one or more specialties via a `TEXT[]` tags field on a `counselor_profiles` table.
- Specialty values mirror `primary_category` enum labels (`academic`, `grief`, `career`, etc.).
- Assignment logic queries available counselors and ranks by specialty overlap with the patient's last known AI report category.

### 9. Ethics & Privacy Gateway
- All users must complete a versioned consent flow before accessing any feature.
- Consent is re-prompted whenever the policy version increments.
- Consent records are permanent and immutable (no DELETE policy).

### 10. Notifications
- In-app alerts for: session scheduled, session reminder, risk alert, general messages.
- Sent by the system (null sender) or by counselors.

## Key Design Decisions

- **Counselors never read raw journals.** All patient voice is mediated through AI summaries to protect patient dignity and reduce counselor bias.
- **AI is a tool, not a decision-maker.** Every AI report has a mandatory counselor evaluation step where recommendations can be approved, modified, or rejected.
- **Admin is the gate for on-request AI reports.** Prevents counselors from mass-generating AI reports without oversight.
- **Specialty-based assignment is soft, not hard.** If no specialty match exists, the system falls back to availability. Patients can always request reassignment.
- **Supabase RLS is the security layer.** No patient can access another patient's data at the database level, regardless of application logic.
